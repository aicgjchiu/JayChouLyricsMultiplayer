寫一個新模式
根據玩家數量選取相同數量的歌曲
以及相同數量的詞

總共有玩家數量的phases,有n個玩家數量，就有n個phase.
每個phase，在時間內，玩家可以多次錄音，再時間截止內上傳最後決定的版本，如果沒有決定好會自動選取最後一個錄音

每一個phase都必須提醒玩家。使用提供的歌詞唱歌，但旋律必須使用聽到的音樂。 音樂可能來自YouTube或其他玩家傳遞過來的錄音

第一個phase 玩家聽YouTube原曲，根據提供的歌詞錄音。
這個一定是不變的

第二到第n-1個phase,都是根據上一個人傳過來的音樂搭配現在的歌詞錄音

n phase,也就是最後一個phase
最後一個人要猜是什麼歌名

結束後就要進結算畫面 ㄧ一播放每首歌傳承下來的結果，以及最後的答案。

假設有n個人，這個結算畫面就要重複n次，因為每首歌的傳承結果都必須讓大家看。
呈現的方式是先播放原本的音樂，然後依序按照順序自動播放每個人的錄音，最後 顯示最後一個人猜得答案是什麼

在這個頁面，在系統呈現的自動播放結束後，使用者可以自由播放任意音頻，直到host決定proceed

播放完一輪的答案後，由房主決定什麼時候proceed，繼續播放下一首歌的傳承結果

每個人必須對每首歌都有參與到，所以你必須合理的安排順序
假設有3首歌， 3個phase，安排的順序就可以是這樣

第一首歌的參與順序
玩家1 ->玩家2->玩家3
歌詞1->歌詞2->歌詞3

第二首歌的參與順序
玩家2->玩家3->玩家1
歌詞3->歌詞1->歌詞2

第三首歌的參與順序
玩家3->玩家1->玩家2
歌詞2->歌詞3->歌詞1

簡單來說，就是每一個phase，每個玩家都有事可以做，且每一個phase做的事情都是跟不同的歌有關。

而且，你必須確保每個玩家每一輪encounter的歌詞都是完全不一樣的
以我剛剛上面的例子為例，每個人每回合遇到要參與的歌跟對應的歌詞是不一樣的

你也必須確保同一首歌傳承下來的歌詞，每一phase都不一樣，以剛剛上面的例子為例，第一首歌，歌詞順序是1->2->3 

phase的持續時間是configurable，房主在開lobby的時候就可以先決定好了
